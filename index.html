1. Introduction

The component model for the Web ("Web Music") consists of three parts:

  1. Elements, which defines playable elements
  2. Resolvers, which defines the music resolver system

1.1. Background

This section is non-normative

Legal streaming music has become the new music business with a lot of different services, such Spotify, Soundcloud, iTunes, AppleMusic etc. To create a fair market for music services we have created this specifications of a set of elements and interfaces that should allow people to share and enjoy music regardless of what service they use.

2. Elements

2.1. The song#elementdef-songReferenced in:2.1. The song elementAttributes (2) (3) (4) (5) (6)3. The IMusicService resolver system (2) element

Thee song element is an universal link to an abstract embedded track independent of media service. This element specifices the title, edit, version, authors and releaese and type of media of content it points to, but not what service it is on.

Embed a song into a web page

<html>
  <head>

  <head>
  <body>
    <h1>A song</h1>
    <song artist="Katy Perry" release="Dark Horse" title="Dark Horse">
  <body>
</html>

2.2. The playlist#elementdef-playlistReferenced in:2.2. The playlist element element

Thee playlist element is an universal link to an abstract embedded track independent of media service. This element specifices the title, edit, version, authors and releaese and type of media of content it points to, but not what service it is on.

Embed a playlist into a web page 
<!DOCTYPE html>
<html>
  <head>
    <title>My page<title>
  <head>
  <body>
    <h1>A song</h1>
    <playlist name="My test playlist">
      <song artist="Katy Perry" album="Dark Horse" name="Dark Horse">
      <song artist="Darude" album="Before the Storm" name="Sandstorm" edit="Radio Edit">
    </playlist>
  <body>
</html>

2.3. Methods

song . play
Play the song

2.4. Attributes

song . isPlaying
Returns if the media is currently playing
The isPlaying attribute must return if the song is currently playing.
song . currentPosition
Returns the current playing position of the media in seconds
The currentPosition attribute must return an positive integer if currently playing, otherwise -1.
song . authors
Returns the current playing position of the media in seconds
The authors must be a string.
song . title
Get sets the title
The title must be a string.
song . release
Get sets the releasee for the media
The release must be a string and represent the name of the release.
song . version
Get sets the version for the song
The version must be a string and represent the name of the release.
song . edit
Get sets the edit for the song. An edit differs from version as it is a cut, like Radio Edit.
The edit must be a string and represent the name of the edit (Eg Radio).

3. The IMusicService#imusicserviceReferenced in:3. The IMusicService resolver system (2) (3) resolver system

In order to be able to play the playlist, the browser must be able to resolve the song elements to authorized music services, and play through their APIs. Music services are integrated into music browser through implementations of the IMusicService class, either in native or JavaScript code, provided as plug-ins for the user agent.

This section is non-normative

To allow playbacks from song elements, songs services can create plugins containing implementions of IMusicService interface in user-installable plugins which resolves the tracks to the particular service and allows instant playback

enum ResolveResult {"available", "unavailable", "playable"};

[OverrideBuiltins]
partial /*sealed*/ interface IMusicService {
attribute DOMString? connected;

ResolveResult resolveAndPlayTrack(HTMLSongElement song);
void seek(Number position);
void stop();
void pause();
};

4. Registering IMusicService resolvers

Registration of an IMusicResolver is done through window.registerMusicService(IMusicResolver resolver)
